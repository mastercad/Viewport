Push-Notification-Zugangsdaten

Warum braucht Blickfang diese Schlüssel?
Electron liefert keine Google-API-Schlüssel mit. Ohne sie verweigert Chromium die Push-Registrierung: Registration failed – push service not available

GOOGLE_API_KEY – Firebase Console → Projekteinstellungen → Allgemein → Web-App → apiKey

CLIENT_ID – Google Cloud Console → APIs & Dienste → Anmeldedaten → OAuth 2.0-Client-ID (Typ: Desktop) → Client-ID

CLIENT_SECRET – dieselbe OAuth-App → Client-Secret

Blickfang leitet diese Zugangsdaten beim Start an Chromium weiter, damit die geladenen Webseiten Push-Notifications empfangen können.
Ohne Konfiguration schlägt die Registrierung fehl: Registration failed – push service not available

Nach dem Speichern die App neu starten, damit die Zugangsdaten wirksam werden.
